# Projects Index

One directory per ongoing project. Three files each (see `example-project/`):

| File | Purpose | Rule |
| :--- | :------ | :--- |
| `description.md` | One-paragraph summary | Injected into the agent's system prompt — keep it current |
| `playbook.md` | Proven state | Architecture, working commands, gotchas. No hypotheses. |
| `scratch.md` | Session work | Append-only. Promote findings to the playbook when proven. |

Manage these from the dashboard's **Projects** tab (every file is editable
there), or let the agent maintain them — its directives tell it to.
